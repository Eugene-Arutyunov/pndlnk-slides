# Модель ДКЦП

## Инварианты обмена

В любом обмене, кто бы чем не обменивался всегда присутствуют следующие элементы:
- Участники обмена (продавец и покупатель)
- То, чем они обмениваются друг с другом (конкретно)
- То, что привело их к обмену
- То как организован обмен по времени
- То как они друг с другом коммуницируют
- То какой смысл имеет обмен для обоих (символический смысл)
*вот тут бы какую-то схему, можно взять иллюстрацию из открытой лекции, но она не в стиле*

---

Модель ДКЦП имеет вот такую форму. Блоки модели соответствуют инвариантам обмена.


Участники обмена:
| | |
| - | - |
| Продавец            | → Атрибуты компании |
| Покупатель | → Клиентский сегмент |

То, чем они обмениваются друг с другом (конкретно)
| | |
| - | - |
| Продавец покупателю | → Артефакт, атрибуты |
| Покупатель продавцу | → Влад клиента |

То, что привело их к обмену:
| | |
| - | - |
| Покупателя | → Ключевой конфликт |
| Продавца | Не учитывается в модели! |

| | |
| - | - |
| То как организован обмен по времени | → Условия обмена |
| То как они друг с другом коммуницируют | → Аргументы |
| То какой смысл имеет обмен для обоих (символический смысл) | → Ценность |

# Ключевой конфликт

Этот блок отвечает за причину нашего представителя клиентского сегмента совершить обмен.

**Корректность КМК**
    1. Определено чьё поведение меняется (представителя КС или другого человека)
    2. Определена деятельность, внутри которой лежит конфликт мотивации
    3. Конфликт мотивации лежит внутри заявленной деятельности
    4. Определён исходный и изменённый образ действия
    5. В ключевом конфликте отстутствует упоминание артефакта компании или прямых конкурентов
    6. Определена общая цель
    7. Понятен ущерб от текущего образа действия
    8. Нет барьеров, сформулированных "через отсутствие"
    9. Для B2B представлены личные и организационные конфликты
    10. Есть свидетельства из исследований о содержании КМК

# Артефакт

Этот блок отвечает за то, что мы передаём клиенту в процессе обмена

**Корректность Артефакта**
    1. Указан класс артефакта (потребительская категория)
    2. Идентифицирован тип (цепочка, мастерская, сеть) артефакта и его составялющих
    3. Среди составляющих артефакта отсутствуют атрибуты

Этот блок описывает свойства артефакта

**Корректность Атрибутов**

	4. Присутствуют уникальные атрибуты (таких, которых нет у категории и/или конкурентов)

# Ценности

Этот блок описывает смысл, значение нашего обмена для сторон

**Корректность ценностей**
    1. Формулировка в 1-2 слова. Если хочется больше – скорее всего это аргумент.
    2. Ценность описывает не физические свойства продукта, а то, что он символизирует. Можно ли потрогать, измерить или сфотографировать эту ценность? Если да - это скорее всего атрибут.
    3. Можно предположить встраивание этой ценности в некоторую иерархию?

# Выводы

- В любом обмене, кто бы чем не обменивался всегда присутствуют тематические блоки, называемые инвариантами
- Модель ДКЦП – это схема на основе инвариантов обмена
- У каждого блока есть критерии корректности, которые позволяют оценить правильно ли мы их заполнили
